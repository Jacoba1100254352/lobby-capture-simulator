# Public Source Data Roadmap

This roadmap translates the May 2026 source-data research into acquisition tasks. It keeps three layers separate:

- direct observed data: source rows that measure a public record directly;
- proxy data: public records that approximate hidden or partly private activity;
- restricted or poor-fit data: useful context that should not be treated as a reproducible public validation source unless an export/license is configured.

## Identifier Spine

The matching spine should prefer stable source-native identifiers:

- LDA: registrant ID, client ID, filing UUID, issue code, covered official position, agency/contact fields.
- FEC/OpenFEC: committee ID, candidate ID, cycle, schedule, transaction ID, spender/payee names.
- IRS and nonprofit records: EIN, organization name, subsection, tax period, return object ID.
- SAM/FPDS/USAspending: UEI, recipient name, awarding agency, PIID, award ID, modification number.
- Regulations.gov and Federal Register: docket ID, document ID, comment ID, object ID, comment-on ID, CFR/title metadata.
- Personnel and access records: official name, agency, position title, date range, employer, committee/hearing ID, Reginfo meeting ID, RIN, requestor, and requestor client.

## Acquisition Sequence

1. Freeze the current 2024 EPA/ENV slice as the baseline paper snapshot.
2. Extend the OpenFEC bridge beyond Schedule E and the implemented electioneering/communication-cost fetchers by adding lobbyist-bundling records, hidden-donor identity evidence where legally observable, and optional FollowTheMoney state/local overlays where available.
3. Broaden the public-financing bridge from the implemented NYC CFB public-funds payment and Seattle Democracy Voucher importers to federal public financing, state programs, and additional local matching-fund or voucher programs where source files can be archived.
4. Broaden the implemented NYC CFB intermediary, IRS EO BMF intermediary-capacity, IRS POFD Form 8872, and ProPublica/IRS Schedule I importers with all IRS 8871/8872 alphabetic slices plus all-organization Form 990 rows for 527, 501(c)(4), 501(c)(6), association, think-tank, and grantmaking intermediaries. Keep donor invisibility as a measured missingness property, not as a filled-in fact.
5. Broaden the procurement bridge from the EPA/USAspending slice, implemented multi-agency top-award bridge, expanded stratified USAspending procurement action panel, archived USAspending bulk transaction summary, procurement benchmark crosswalk, and optional separate SAM.gov Contract Awards panel into action-history coverage with exclusions, action-level modifications, award-action dates, protest links, and firewall coverage for procurement firewalls, ex-post modifications, concentration, and single-bid or price-only risks. Use `make first-wave-procurement-source-acquisition` to regenerate the official-source acquisition handoff for the required `sam-fpds-action-history-crosswalk`, `gao-protest-overlay`, `sam-exclusion-overlay`, `procurement-firewall-overlay`, and offer-count/competition enrichment products. Use `make gao-protest-overlay-candidates` to refresh the GAO protest overlay as a candidate-only official-feed review queue before source-page adjudication; do not remove candidate markers until filed dates, outcome/issue coding, and PIID/UEI or reviewed vendor linkage have been confirmed. The no-key USAspending bulk route has cleared full acquisition for configured agency/time strata; rerun `make usaspending-transaction-download-strata` and `python3 scripts/audit-usaspending-transaction-download-strata.py --download` only when refreshing or expanding that archive. The SAM no-quota path should set `SAM_CONTRACT_AWARDS_LIVE_CSV` or `SAM_CONTRACT_AWARDS_LIVE_URL` to a downloaded representative Contract Awards/DataBank CSV/JSON/ZIP export, then run `make sam-procurement-refresh` so the export audit must report `promotion-readiness=candidate` before snapshot promotion. The keyed-refresh path should use `SAM_CONTRACT_AWARDS_EXTRACT_MODE=1` for larger asynchronous JSON/CSV extracts, or multiple agency, department-code, or PIID-subtier filters plus non-adjacent `SAM_CONTRACT_AWARDS_OFFSET_STARTS` page indexes for bounded synchronous samples. If the initial extract-token request times out before a `presignedUrl` is returned, verify that the SAM.gov key/account has Contract Awards API access, required role permissions, allowed network origin, and any 429 `nextAccessTime` before treating the failure as a model/data result. Keep `SOURCE_FETCH_CURL_FALLBACK=1` enabled when SAM responds to curl but hangs under the Python request path.
6. Expand the LDA-derived revolving-door panel with documented source exports from LegiStorm, OGE, ProPublica appointee records, FACA, or manually archived public personnel data so covered-position headcount can be separated from post-employment movement and access intensity.
7. Broaden the implemented Reginfo.gov EO 12866 meeting importer with actor/entity linkage to LDA clients, docket IDs, FEC committees, nonprofit intermediaries, procurement vendors, and rulemaking outcomes; then add FACA and House witness disclosure panels for expert-access and intermediary venue-shifting diagnostics.
8. Run `make procurement-refresh-readiness` before any manual export or keyed SAM.gov refresh to confirm the configured acquisition path, fallback policy, and claim boundary without spending API quota.
9. Run `make sam-contract-awards-preflight` to make a one-row redacted SAM.gov Contract Awards access/quota check. If it reports `quota_blocked`, wait for the reported UTC reset before running the keyed live snapshot, or use the manual SAM export path or no-key USAspending transaction-download strata route instead. If it reports `ok`, run the representative refresh settings immediately.
10. Rerun `make snapshot-2024-env source-moments procurement-benchmark-crosswalk validate calibration-queue procurement-refresh-readiness paper-artifacts-check` and commit regenerated artifacts.

`make source-panel-inventory` should be run after each source refresh. It classifies direct dark-money, outside-spending, public-financing, intermediary, IRS 527, revolving-door, OIRA meeting-log, procurement-identifier, procurement-concentration, and procurement-modification panels as usable, thin, warning-level, fixture-only, or missing so model claims cannot silently outrun source coverage.

## Claim Discipline

- LDA, FEC, NYC CFB, Seattle Democracy Voucher, Regulations.gov, Federal Register, Reginfo.gov EO 12866 meetings, USAspending, SAM/FPDS, IRS EO BMF, IRS 8871/8872, and filed Form 990 data can support observed distribution claims within their coverage limits.
- OpenSecrets, LegiStorm, ProPublica, FACA, House witness disclosures, and OGE records can strengthen bridge panels, but licensing, coverage, matching, and missingness must be disclosed.
- 501(c)(4) and 501(c)(6) donor identities are often not public. The simulator should model this as opacity and traceability; the IRS EO BMF dark-money bridge is an opaque-capacity proxy, while public Schedule I nonprofit-routing rows are transfer evidence rather than direct donor-identity evidence.
- Cross-venue detection, participation protection, and speech-restriction risk are portfolio-design metrics until source panels are built to observe coverage, false positives, and enforcement practice.
