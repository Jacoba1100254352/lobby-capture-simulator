# Validation Plan

This project uses external data to constrain plausible ranges, not to claim causal estimates.

Initial validation targets:

- LDA filings: lobbying spending distributions, issue-domain mix, registrant/client concentration, covered official fields.
- FEC data: large donor dependence, PAC/independent expenditure pressure, lobbyist bundling, outside-spending concentration.
- Federal Register and Regulations.gov: rulemaking duration, docket/comment volume, agency/topic mix.
- Voteview and govinfo bill status: legislative attrition and coalition plausibility for legislative arenas.
- NYC public matching funds, Seattle democracy vouchers, and related programs: voucher participation, public-funds share, donor-base broadening.
- USAspending: procurement-recipient concentration and agency spending exposure.
- SAM/FPDS: contract-award identifiers, UEI/PIID matching, modifications, exclusions, and procurement-firewall stress tests.
- IRS EO BMF, IRS 8871/8872, and Form 990 XML: nonprofit, 527, association, think-tank, and intermediary panels, while preserving public-donor missingness.
- FACA, House witness disclosures, OGE, OpenSecrets, LegiStorm, and ProPublica-style panels: access, expert, intermediary, and revolving-door bridges where coverage and licensing allow.

The implementation stores benchmark metadata in `data/calibration/empirical-benchmarks.csv`, normalized fixture rows in `data/fixtures/`, source-native JSON parser fixtures in `data/fixtures/source-native/`, and the committed paper snapshot under `data/snapshots/2024-env/normalized/`. Report generation defaults to that committed snapshot so ignored `data/raw/` live-fetch outputs cannot alter paper artifacts unless `LOBBY_CAPTURE_CALIBRATION_DIR` is set explicitly.

The Deep Research reports are distilled into `docs/research/research-synthesis.md` and `data/calibration/parameter-map.csv`. The parameter map is the preferred place to add new empirical moments because it records a model metric, low/mid/high prior, evidence class, source report, model target, and implementation status.

Current calibration links:

- `normalized-lda-lobbying.csv` constrains issue funding scale, issue mix, and the source timing input used for the lobbying visibility-lag component. The report applies a public-visibility transform so archived LDA source lag is not compared directly to current public-visibility benchmarks. FEC, public-financing, and dark-money rows carry a separate campaign visibility-lag component; the report-level `averageDisclosureLag` remains an operational blend rather than a source-specific validation target.
- `normalized-fec-campaign-finance.csv` constrains donor concentration, Schedule E outside-spending pressure, traceability, and bounded OpenFEC electioneering/communication-cost coverage. OpenFEC electioneering and communication-cost rows are included in the pinned snapshot as observed electoral-communication bridge rows; `normalized-public-financing.csv` and `normalized-dark-money.csv` carry separate public-financing, opaque-capacity, and bounded Schedule I nonprofit-routing bridge rows.
- `normalized-regulatory-dockets.csv` constrains docket volume, comment authenticity, template saturation, and technical-claim credibility.
- `reports/source-moments.csv` records direct top-k concentration, traceability, non-proxy dark-money routing row coverage, direct dark-money visibility, Schedule E outside spending, electoral-communication coverage, public financing, comment-record, procurement bridge, intermediary, and revolving-door moments from the current snapshot and fixture baselines.

These files are plausibility inputs. They should not be cited as causal estimates, and paper claims should distinguish three layers: observed distribution, model calibration choice, and simulated mechanism.

The fetch scripts now have two modes:

- default mode copies tracked deterministic fixtures into `data/raw/` for snapshot preparation and local experiments;
- `--live` mode normalizes an explicit CSV path or URL into the same schema and fails if required columns are missing;
- source-native `--live` mode queries LDA, OpenFEC, Regulations.gov, Federal Register, USAspending, NYC CFB, Seattle Democracy Voucher, IRS EO BMF, ProPublica/IRS Schedule I, IRS POFD Form 8872, or LDA-derived covered-position revolving-door records when no explicit CSV/URL is supplied.

Expected live-mode environment variables are `LDA_LIVE_CSV` or `LDA_LIVE_URL`, `FEC_LIVE_CSV` or `FEC_LIVE_URL`, `REGULATORY_LIVE_CSV` or `REGULATORY_LIVE_URL`, and `SAM_CONTRACT_AWARDS_LIVE_CSV` or `SAM_CONTRACT_AWARDS_LIVE_URL` for explicit-source normalization.

The source-native normalizers are distributional bridges. `scripts/test-source-fetchers.py` verifies representative LDA, OpenFEC contribution, OpenFEC electioneering, OpenFEC communication-cost, Regulations.gov, Federal Register, USAspending, SAM.gov Contract Awards, NYC CFB, Seattle Democracy Voucher, IRS EO BMF, ProPublica/IRS Schedule I nonprofit-routing, and IRS POFD Form 8872 payloads against exact normalized rows before any live API output is used as a paper snapshot.

The source acquisition roadmap in `docs/source-data-roadmap.md` records the intended identifier spine and distinguishes direct observed sources from proxy and restricted overlays. New source panels should be added there before their moments are promoted into `data/calibration/parameter-map.csv`.

`make source-moments` writes `reports/source-moments.csv` plus `reports/source-moments.md`. It compares the current 2024 EPA/ENV normalized snapshot against the deterministic fixture baseline on source-level moments such as top-client share, top-donor share, amount-weighted traceability, non-proxy direct dark-money routing rows, direct dark-money visibility, Schedule E outside-spending source share, electioneering and communication-cost row coverage, public-financing program count, voucher rows, matching-fund rows, public-financing source share, comment-volume concentration, procurement concentration, competition, and modification panels, UEI/PIID coverage, intermediary donor disclosure, intermediary political-spend share, IRS 527/Form 8872 rows and political-spend share, and revolving-door source-confidence diagnostics. Procurement source moments keep SAM.gov Contract Awards, the balanced USAspending action panel, and the no-agency-filtered national USAspending action panel separate. They prefer archived SAM rows for concentration if present, otherwise use the national USAspending action panel for agency and recipient concentration diagnostics, while retaining the balanced action panel for modification incidence. The top-award bridge remains a separate high-value diagnostic, award rows with competition fields support single-bid and limited-competition diagnostics, and transaction-row, distinct-award, and amount-weighted modification diagnostics remain separate when either `usaspending-procurement-actions.csv` or `sam-contract-awards.csv` is present. Award rows remain a fallback and are flagged as a coverage warning when they are dominated by initial awards. `make source-panel-inventory` then writes `reports/source-panel-inventory.csv` and `.md`, classifying whether direct dark-money, outside-spending, electoral-communication, public-financing, intermediary, IRS 527, revolving-door, and procurement panels are usable, thin, warning-level, fixture-only, or missing.

`make validate` compares committed report snapshots against both `data/calibration/empirical-benchmarks.csv` and implemented rows in `data/calibration/parameter-map.csv`. It writes `reports/validation-summary.csv` plus `reports/validation-summary.md`, including counts by evidence class and scope notes for scenario-specific benchmarks such as public-financing uptake, voucher participation, campaign-finance concentration, opaque electoral routing, shadow-lobbying substitution, cooling-off venue shifting, comment-authentication duplicate compression, reporting-error detection, campaign sanction incidence, regulator backlog, and procurement exposure. The lobbying reporting-error benchmark is validated against `reportingErrorDetectionRate`, not the broader `detectionRate` capture-enforcement diagnostic. The campaign penalty benchmark is validated against `campaignSanctionIncidence`, not the broader per-contest `sanctionRate`. The public-financing candidate-uptake benchmark is scoped to full candidate-side public-financing regimes; partial public-interest representation, countervailing-representation, civil-liberties, high-evasion stress, and model-mode comparison rows are kept separate because they do not represent clean candidate adoption tests. The regulator backlog benchmark is scoped to referral/workload-bearing enforcement and substitution rows, excluding disabled model-mode comparisons, procurement-specific rows, electoral leakage rows, visible-ban rows, cooling-off-only rows, machine-readable meeting-log rows, and other low-workload rows where unresolved referral pressure is not the modeled target. The high-end dark-money/Super PAC routing benchmark is scoped to explicit opaque electoral-routing stress scenarios with high hidden dark-money pressure; the broader shadow-lobbying benchmark separately covers non-electoral advisory, intermediary, venue-shift, and hard-budget substitution while keeping venue-detection rows out of the shadow-lobbying family. Influence-preservation checks exclude no-substitution and single-channel rows and apply only to substitution-active scenarios. Cooling-off venue-shift checks exclude rows where cross-venue detection deliberately suppresses the venue-shift mechanism. Synthetic bounded diagnostics such as observed capture and cross-venue detection validate against their normalized 0-1 ranges; saturation concerns are handled in paper tables and interpretation instead of being classified as calibration failures. The duplicate-compression benchmark is scoped to comment-authentication, public-advocate, rulemaking-integrity, full-bundle, and ablation scenarios rather than every docket-bearing run, because the parameter range describes deduplication tooling rather than ordinary low-reform comment handling. Source-moment rows can also be classified as `source_gap` when the available panel is too narrow, too proxy-backed, or mapped to an incompatible benchmark denominator. The USAspending procurement bulk summary is now accompanied by `reports/procurement-benchmark-crosswalk.md`, which remaps top-contractor concentration and ex-post modification ranges to the observed aggregate, agency, award-type, action-row, distinct-award, and amount-weighted denominators. It also writes `reports/substitution-audit.csv` and `reports/substitution-audit.md`, which distinguish distortion failures, hidden-capture warnings, hidden-influence warnings, substitution warnings, and channel-shift tradeoffs. Pure movement into a different channel without a distortion increase is classified as a channel-shift tradeoff rather than a clean failure. Network metrics are included as synthetic path diagnostics: they currently validate bounds and mechanism direction, not observed network reconstruction. Current benchmark rows are deliberately broad plausibility bands; misses and source gaps should be read as calibration work queues, not model falsification.

`make portfolio` writes `reports/lobby-capture-portfolio.csv` plus `reports/lobby-capture-portfolio.md`. It screens reform bundles by total influence distortion, hidden capture, substitution risk, administrative burden, network opacity, legitimate-advocacy chill, speech-restriction risk, cross-venue detection, and participation protection.

`make calibration-queue` writes `reports/calibration-queue.csv` plus `reports/calibration-queue.md`. It classifies misses and partial overlaps as model tuning, metric splits, direct source moments, scenario coverage, scenario-family splits, scale alignment, or benchmark review. Scenario-family splits mean the scoped report rows already span baseline, stress, and extreme-stress behavior, so the next step is to split the validation family or paper interpretation before treating one benchmark range as a single calibration target.

`make calibration-readiness-audit` writes `reports/calibration-readiness.csv` plus `reports/calibration-readiness.md`. It separates P0/P1/P2 blockers for calibrated policy-simulation claims from P3 benchmark-scope and scenario-coverage work. This makes clear when the manuscript is cleared as a mechanism-model article with bounded empirical bridges while calibrated policy claims remain blocked.

`make submission-readiness-audit` writes `reports/submission-readiness.csv` plus `reports/submission-readiness.md`. It is the final synthesis layer for the review bundle: mechanism-model readiness should be `ready`, empirical-bridge scope should remain `bounded`, calibrated-policy claims should remain `blocked`, and the overall posture should be `ready_for_mechanism_review`. This is a review-bundle gate, not a claim that causal policy calibration or final venue acceptance has been achieved. The separate `reports/final-human-readthrough.md` checklist records the manual editorial read-through and live Regulation & Governance author-page refresh needed for final journal submission; the audit only treats those portions as complete when the checklist is signed and the live-page fields are completed for the current release tag.

`make doi-deposit-package` writes ignored `dist/lobby-capture-doi-deposit-package.zip`, `dist/doi-deposit-package-manifest.{json,md}`, and `dist/doi-deposit-package-checksum.{csv,json,md}` after the Wiley package, archive-handoff manifest, Wiley upload-form audit, and Regulation & Governance guideline audit are built. `make doi-deposit-readiness-audit` writes `reports/doi-deposit-readiness.csv` plus `reports/doi-deposit-readiness.md` after the DOI deposit package is built. It checks release metadata, primary release assets, release-machine checksum files, DOI package integrity, claim-boundary posture, DOI-record status, final human-read-through status, and live author-page refresh status. This is a release-aid gate only: it can show that the asset set is ready for DOI deposition, but it must not be read as proof that a DOI has been minted unless the DOI appears in the citation/deposit metadata and final read-through record.

`make external-finalization-checklist` writes ignored `reports/external-finalization-checklist.csv` plus `reports/external-finalization-checklist.md`. It sources `.env`, then consolidates DOI readiness, post-release GitHub asset verification, Zenodo target/token/draft/upload state, DOI metadata-record state, human read-through state, live author-page-refresh state, and any configured SAM.gov Contract Awards export URL or CSV. It is intentionally excluded from the deterministic paper gate because it can reflect private credentials and live external services. Use it as the local handoff surface before DOI publication, journal submission, or a SAM/FPDS refresh, not as evidence that any of those external steps have been completed. The checklist separates DOI/journal finalization posture from source-refresh posture; a blocked SAM export means the candidate export should not be promoted into a refreshed snapshot, not that the current released review bundle is invalid. When a SAM export audit is present, the checklist carries the hard blocker names and raw action-date, solicitation-date, and amount-field coverage so a solicitation-like export is not mistaken for a promotable action-history panel.

`make github-release-asset-audit` is a networked post-release check, not part of the offline artifact gate. It queries the current release with the GitHub CLI and compares uploaded asset sizes and SHA-256 digests against the local release-machine checksum manifests and generated release-aid reports. Use it after creating or replacing GitHub release assets and before copying checksums into a DOI deposit record.

`make zenodo-deposit-preflight` writes `reports/zenodo-deposit-preflight.csv`, `reports/zenodo-deposit-preflight.md`, and ignored `dist/zenodo-deposit-metadata.json`. `make latex-log-audit` writes `reports/latex-log-audit.csv` plus `reports/latex-log-audit.md`. `make causal-calibration-targets` writes `reports/causal-calibration-targets.csv` plus `reports/causal-calibration-targets.md`. This audit names the independent causal estimands, source families, current support, and clearance criteria required before calibrated policy-simulation claims can clear. `make first-wave-causal-protocols` writes `reports/first-wave-causal-protocols.csv`, `reports/first-wave-causal-protocols.md`, and `paper/tables/first_wave_causal_protocols.tex`. It operationalizes the first-wave causal targets by specifying the unit of analysis, treatment or shock, comparison design, primary outcomes, linkage keys, minimum source products, falsification checks, sensitivity checks, threat model, and claim-upgrade boundary for each protocol. This report is a pre-results design artifact: it does not clear calibrated policy-simulation claims, but it prevents future source promotion from happening without a documented identification and validity plan.

`make first-wave-source-product-templates` writes acquisition templates under `docs/source-product-templates/first-wave/`. These templates provide headers, design-note prompts, and a manifest for the required products, but they are not source evidence and cannot satisfy the production gate. `make first-wave-source-products` writes `reports/first-wave-source-products.csv` plus `reports/first-wave-source-products.md`. It translates the first-wave protocols into concrete source-product schemas under `data/calibration/first-wave/`, with expected paths, template paths, required columns or text terms, acceptable source families, minimum-row checks, field-level quality checks, product-level semantic gates, validation rules, claim boundaries, and next acquisition actions. This schema/acquisition gate is intentionally separate from source moments: public surfaces can be usable as mechanism anchors while still lacking the event panels, crosswalks, overlays, comment corpora, and entity-resolution products needed for causal estimation. The semantic gates prevent tiny or structurally incomplete source files from clearing causal-readiness checks merely because they contain the right headers.

`make first-wave-source-readiness` writes `reports/first-wave-source-readiness.csv` plus `reports/first-wave-source-readiness.md`. `make sam-contract-awards-export-audit` reads `SAM_CONTRACT_AWARDS_LIVE_CSV` or `SAM_CONTRACT_AWARDS_LIVE_URL` from the ignored local environment and writes ignored operational reports under `reports/sam-contract-awards-export-audit.*`. `make usaspending-transaction-download-strata` writes ignored operational reports under `reports/usaspending-transaction-download-strata.*`. `make sam-procurement-refresh` is the guarded one-command wrapper for that refresh. It prefers a configured downloaded export, otherwise requires an `ok` preflight before running the keyed SAM.gov Contract Awards extract path. If the preflight reports `quota_blocked`, it exits before running the live snapshot or regenerating paper artifacts. A successful run regenerates the live snapshot and then runs `make paper-artifacts-check` so source rows, reports, tables, figures, PDFs, and the submission package stay synchronized.
